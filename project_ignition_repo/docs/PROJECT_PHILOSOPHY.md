# Project Philosophy

Project Ignition exists to reduce friction between an idea and making progress.

## Guiding Principles

### Local-first

The user's data belongs to the user.

### Self-hosted by default

The primary deployment target is a personal PC, NAS, homelab server, or private VPS.

### Database-driven

If a user may want to change it, the user should be able to change it through the UI.

### Modular

Features should be added as modules, not tangled one-off additions.

### Maker-focused

This is not generic corporate project management software.

### Progress over perfection

The app should help the user do the next useful thing.

### Offline-capable

Core functionality should work without internet access.

## Product Question

Before adding a feature, ask:

> Does this reduce friction between an idea and completing it?
